# ANDYBOT — Capacidades y límites

> Mapa honesto de qué puede y qué no puede hacer Andy. Evita expectativas falsas.

## SÍ puede

| Capacidad | Cómo |
|-----------|------|
| Consulta por voz/texto en Telegram | Whisper transcribe → GPT responde |
| Recordar estado de un proyecto | Lee ESTADO.md del repo |
| Recordar decisiones y su porqué | Lee DECISIONES.md |
| Listar tareas pendientes | Lee TAREAS.md |
| Traducir términos técnicos | Lee GLOSARIO.md |
| Interpretar imágenes (esquemas, logos) | GPT-4o con visión |
| Primeras ideas para un prospecto | Razona sobre el caso + biblioteca |
| Sugerir flujos reutilizables | Lee biblioteca/flujos-reutilizables/ |

## NO puede (y lo dice con honestidad)

| No puede | Quién lo hace |
|----------|---------------|
| Escribir código complejo | CC |
| Depurar flujos n8n rotos | Codex / Fable |
| Decidir arquitectura | Fable |
| Auto-mejorarse solo | Nadie: tú agregas conocimiento, él lo lee |
| Vectorizar un logo (trazado técnico) | Herramienta de diseño aparte |
| Guardar o repetir contraseñas | Nadie: van en Bitwarden |
| Inventar tarifas sin datos | Se resuelve con Fable cuando haya caso real |

## La verdad sobre "que aprenda"

Andy no evoluciona solo. Lo que llamas "aprender" funciona así:
1. Fable resuelve un problema difícil.
2. La solución se guarda en biblioteca/aprendizajes/.
3. La próxima vez, Andy LEE ese archivo y aplica la solución.

Andy parece más listo cada mes no porque cambie, sino porque su biblioteca crece.
El motor del aprendizaje eres tú + los cerebros potentes. Andy es la memoria.
